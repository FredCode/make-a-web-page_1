Introduction to Web & HTML

Basics

A web page is a text document written in a language called HTML. Web browsers read these documents, and then interpret and display them.

HTML stands for hypertext markup language. More details here.

HTML = language = syntax + rules.

A tag links to Elements that form a tree ; DOM (Document Object Model) is a standard convention for representing and interacting with elements in HTML.

An element is created from the opening tag to the closing tag and everything in between is considered content.

Content can be empty/text/ or another element.All tags have attributes & values.

Coding happens when programmers write text in a language that a computer can understand. The computer can then follow the instructions the programmer wrote.

Programmers need to write exactly the way a computer understands (also known as writing with correct "syntax").

HTML Vocabulary

Tag:

An HTML tag is always contained within angled brackets. Most tags have an opening tag and a closing tag. Some tags (called "void" tags) do not require a closing tag.

Element:

An HTML element refers to everything within a set of opening and closing tags.

Attribute:

This is a property of an HTML element. For example, Udacity

Inline vs Block Elements:

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Introduction to CSS

Basics

What is CSS?

CSS stands for "Cascading Style Sheets". It is a language that helps you define the style of the page.

Similarly as HTML, CSS = specific syntax + rules and allows you to change how elements look on the page (font size, colour, background, position etc.). In CSS, the most specific rule applies.

CSS Boxes

We define boxes with Div tags and create as many Div tags as boxes. You need to use class names so as to know the purpose of each box just by glancing at the code.

When you read an HTML document as text, there are changes of indentations: the more indented an element is, the more deeply nested

The concept of Inheritance

Inheritance is a key feature in CSS, it relies on the ancestor-descendant relationship to operate. Inheritance is the mechanism by which properties are applied not only to a specified element, but also to its descendants.

Example of CSS Rules

Change background = background-color: (A) for attribute

Add style to certain sections (h1), use div class

How to include CSS Styling in the web page

Method 1 consists in writing CSS in the head of the HTML. A reasonable approach for small projects only.

Method 2 consists in linking your HTML to a separate CSS file. Much better for larger projetcs.

Method 3 consists in writing the style inline with the HTML. Considered not ideal because of code repetition.

Advanced

Advanced details on CSS Boxes

Center: Content & nucleus

Padding: Clears an area around the content, affected by background colour of the box. Space inside the box that protects the content directly.

Border: goes around the padding & content. Inherited from the colour property of the box.

Margin: Outer layer, clears an area around the border. No background colour, space between boxes.

Pixels VS %: You can set sizes of the box as pixels or percentages. As a percentage, its size will change based on large /small the screen is while in pixels, size wonâ€™t change.

Boxing & Positioning

Each box has 4 components.There are two techniques you can use to help deal with sizing issues:

  • Set sizes in terms of percents rather than pixels.
  • Set the box-sizing attribute to border-box for every element.

Regarding Box positioning:

  • Divs are block elements (as opposed to inline), so by default they take up the entire width of a page.
  • Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other.
